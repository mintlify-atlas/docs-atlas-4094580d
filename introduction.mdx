## What are B-cos Networks?

B-cos Networks are a family of inherently interpretable deep learning models for computer vision. Unlike traditional neural networks that require post-hoc explanation methods, B-cos models are designed from the ground up to be explainable, providing transparent insights into their decision-making process.

These models achieve state-of-the-art accuracy on ImageNet while being able to generate visual explanations that highlight which parts of the input image contributed to the prediction.

<Note>
  B-cos v2 models are more efficient and easier to train than the original v1 models, with a large number of pretrained models available for immediate use.
</Note>

## Key Features

### Inherent Interpretability

B-cos Networks use B-cos alignment principles to create models where explanations are built into the architecture itself. Each model has a `.explain()` method that generates visual explanations showing:
- Which regions of the image influenced the prediction
- How different color channels contributed to the decision
- Contribution maps for spatial locations

### Wide Model Zoo

The library provides pretrained models across multiple architectures:
- **ResNets**: ResNet-18, 34, 50, 101, 152, ResNeXt-50
- **DenseNets**: DenseNet-121, 161, 169, 201
- **ConvNeXt**: Tiny and Base variants
- **Vision Transformers**: Simple ViT and ViT-C in various sizes (Tiny, Small, Base, Large)
- **VGG**: VGG-11

All models are pretrained on ImageNet and ready for inference or fine-tuning.

### Easy Integration

Models can be loaded directly via `torch.hub` with no additional dependencies:

```python
import torch
model = torch.hub.load('B-cos/B-cos-v2', 'resnet50', pretrained=True)
```

Each model comes with:
- An inference transform (`model.transform`)
- An `.explain()` method for generating visual explanations
- Methods to access the classifier and feature extractor for fine-tuning

### Advantages over Post-hoc Methods

1. **No Additional Computation**: Explanations are generated through a single forward pass with gradients, unlike methods like LIME or RISE that require multiple evaluations
2. **Consistent Explanations**: Built into the model architecture, not dependent on external explanation algorithms
3. **Fine-grained**: Pixel-level explanations that preserve spatial resolution
4. **Fast**: Real-time explanation generation suitable for interactive applications

## Architecture Highlights

B-cos Networks replace standard layers with B-cos equivalents:

- **BcosConv2d**: Dynamic linear convolution layers with normalized weights
- **BatchNorm2dUncentered**: Uncentered batch normalization without bias
- **Explanation Mode**: Special forward pass mode that enables gradient-based explanations

These components work together to ensure that the gradient with respect to the input provides meaningful explanations.

<Warning>
  B-cos models expect 6-channel input (RGB + inverse RGB channels). The `model.transform` handles this automatically when preprocessing images.
</Warning>
