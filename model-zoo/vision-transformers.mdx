# Vision Transformer Models

This page documents all Vision Transformer (ViT) models in the B-cos Model Zoo. The collection includes both B-cos variants (inherently interpretable) and standard variants (non-B-cos baselines for comparison).

<Note>
All ViT models require the `einops` package. Install it with: `pip install einops`
</Note>

## B-cos Simple ViT Models

Simple ViT models use standard patch-based tokenization with 16×16 patches and 224×224 input resolution. These models follow the "Better plain ViT baselines" architecture.

### Usage Example


### Key Features

- **Pure transformer architecture**: No convolutional components
- **16×16 patch size**: Standard ViT tokenization
- **B-cos interpretability**: Built-in explanation generation
- **Multiple scales**: From Tiny (5.8M) to Large (178.79M) parameters

## B-cos ViT-C Models

ViT-C (ViT with Convolutional stem) models use early convolutions before the transformer layers, improving performance by learning better initial representations. After the convolutional stem, these models process 1×1 patches at 14×14 resolution.

### Usage Example

```python
import torch

# Load B-cos ViT-C Base
model = torch.hub.load('B-cos/B-cos-v2', 'vitc_b_patch1_14', pretrained=True)
```

### Key Features

- **Convolutional stem**: Processes images with early convolutions
- **1×1 patch size**: After stem, operates on 14×14 feature maps
- **Better accuracy**: Consistently outperforms Simple ViT counterparts
- **Hybrid architecture**: Combines CNN and transformer strengths
- **Parameter efficient**: Similar sizes to Simple ViT but higher accuracy

## B-cos vs Standard Comparison

### Simple ViT: B-cos vs Standard

| Size | B-cos Acc | Standard Acc | Difference |
|------|-----------|--------------|------------|
| Tiny | 59.96% | 70.23% | -10.27% |
| Small | 69.25% | 74.47% | -5.22% |
| Base | 74.41% | 75.30% | -0.89% |
| Large | 75.06% | 75.71% | -0.65% |

### ViT-C: B-cos vs Standard

| Size | B-cos Acc | Standard Acc | Difference |
|------|-----------|--------------|------------|
| Tiny | 67.26% | 72.59% | -5.33% |
| Small | 74.50% | 75.76% | -1.26% |
| Base | 77.15% | 76.79% | +0.36% |
| Large | 77.78% | 77.87% | -0.09% |

### Key Insights

- **Accuracy gap decreases with model size**: Larger B-cos models approach or match standard model performance
- **ViT-C closes the gap**: Convolutional stems help B-cos models compete better
- **Base and Large models**: B-cos ViT-C Base/Large achieve competitive accuracy while maintaining interpretability
- **Interpretability trade-off**: Smaller models show larger accuracy differences, but gain inherent interpretability

## Model Selection Guide

### Best B-cos ViT Accuracy

1. **vitc_l_patch1_14**: 77.782% (167.44M params)
2. **vitc_b_patch1_14**: 77.152% (81.37M params)
3. **simple_vit_l_patch16_224**: 75.060% (178.79M params)

### Best Parameter Efficiency (B-cos)

1. **vitc_ti_patch1_14**: 67.260% with 5.32M params (12.64% acc/M)
2. **simple_vit_ti_patch16_224**: 59.960% with 5.80M params (10.34% acc/M)
3. **vitc_s_patch1_14**: 74.504% with 20.88M params (3.57% acc/M)

### Best for Research

- **B-cos and Standard pairs**: Compare interpretable vs standard models
- **vitc_b_patch1_14** + **standard_vitc_b_patch1_14**: Mid-size with strong performance
- **simple_vit_s_patch16_224** + **standard_simple_vit_s_patch16_224**: Good balance of size and accuracy

### Recommended for Production

- **vitc_b_patch1_14**: Best accuracy-to-parameter ratio with interpretability
- **vitc_s_patch1_14**: Smaller footprint, good accuracy
- **simple_vit_s_patch16_224**: When pure transformer architecture is preferred

## Common Usage Patterns

### Basic Inference

```python
import torch
from PIL import Image

model = torch.hub.load('B-cos/B-cos-v2', 'vitc_b_patch1_14', pretrained=True)
model.eval()

# Load and transform image
img = model.transform(Image.open('image.jpg'))
img = img[None]  # Add batch dimension

# Forward pass
with torch.no_grad():
    output = model(img)
    pred = output.argmax(dim=1)
print(f"Predicted class: {pred.item()}")
```

### Generate Explanations

```python
import torch
from PIL import Image
import matplotlib.pyplot as plt

model = torch.hub.load('B-cos/B-cos-v2', 'simple_vit_b_patch16_224', pretrained=True)
model.eval()

# Prepare image with gradients
img = model.transform(Image.open('image.jpg'))
img = img[None].requires_grad_()

# Generate explanation
expl_out = model.explain(img)

print(f"Prediction: {expl_out['prediction']}")
plt.imshow(expl_out['explanation'])
plt.title('B-cos Explanation')
plt.axis('off')
plt.show()
```

### Fine-tuning for Custom Tasks

```python
import torch
import torch.nn as nn

# Load pretrained model
model = torch.hub.load('B-cos/B-cos-v2', 'vitc_s_patch1_14', pretrained=True)

# Get components
feature_extractor = model.get_feature_extractor()
classifier = model.get_classifier()

# Replace classifier for your task
num_classes = 10  # Your dataset classes
new_classifier = nn.Linear(classifier.in_features, num_classes)

# Create new model
class CustomViT(nn.Module):
    def __init__(self, feature_extractor, classifier):
        super().__init__()
        self.features = feature_extractor
        self.classifier = classifier
    
    def forward(self, x):
        x = self.features(x)
        x = self.classifier(x)
        return x

finetuned_model = CustomViT(feature_extractor, new_classifier)

# Freeze feature extractor (optional)
for param in feature_extractor.parameters():
    param.requires_grad = False
```

### Comparing B-cos vs Standard

```python
import torch
from PIL import Image

# Load both variants
bcos_model = torch.hub.load('B-cos/B-cos-v2', 'simple_vit_s_patch16_224', pretrained=True)
std_model = torch.hub.load('B-cos/B-cos-v2', 'standard_simple_vit_s_patch16_224', pretrained=True)

bcos_model.eval()
std_model.eval()

# Compare predictions
img = bcos_model.transform(Image.open('image.jpg')).unsqueeze(0)

with torch.no_grad():
    bcos_out = bcos_model(img)
    std_out = std_model(img)

print(f"B-cos prediction: {bcos_out.argmax()}")
print(f"Standard prediction: {std_out.argmax()}")

# Only B-cos models support explanation
img_with_grad = img.requires_grad_()
expl = bcos_model.explain(img_with_grad)
print(f"Explanation available: {expl['explanation'].shape}")
```

## Architecture Details

### Simple ViT Architecture

1. **Patch Embedding**: 16×16 patches from 224×224 images → 196 tokens
2. **Transformer Encoder**: Multi-head self-attention + MLP blocks
3. **Classification Head**: Global average pooling + linear classifier

### ViT-C Architecture

1. **Convolutional Stem**: Multiple conv layers reducing spatial size
2. **Feature Maps**: 14×14 spatial resolution
3. **Patch Embedding**: 1×1 patches (already processed by stem)
4. **Transformer Encoder**: Multi-head self-attention + MLP blocks
5. **Classification Head**: Global average pooling + linear classifier

### Model Sizes

| Size | Embed Dim | Depth | Heads | MLP Ratio |
|------|-----------|-------|-------|----------|
| Ti (Tiny) | 192 | 12 | 3 | 4 |
| S (Small) | 384 | 12 | 6 | 4 |
| B (Base) | 768 | 12 | 12 | 4 |
| L (Large) | 1024 | 24 | 16 | 4 |
